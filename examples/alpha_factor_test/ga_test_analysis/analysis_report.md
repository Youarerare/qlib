# 遗传算法因子搜索功能分析报告

## 1. 代码审查结果

### 1.1 遗传算法整体流程评估

**✅ 正确部分：**
- 种群初始化：随机生成表达式树，结构合理
- 选择机制：使用锦标赛选择（tournament selection），实现正确
- 交叉操作：实现了子树交换，逻辑基本正确
- 变异操作：包含多种变异策略（替换子树、改变窗口、改变字段、改变操作符）
- 精英保留：保留了每代最好的个体（elite_count = max(2, population_size // 10)）

**❌ 存在问题：**

1. **变异操作中的正则表达式导入位置错误**
   - 文件：`ga_search.py` 第253行
   - 问题：`import re as _re` 放在类定义中间，不符合Python规范

2. **交叉操作的子树重建逻辑过于简化**
   - `_rebuild_from_parts` 方法（第246-250行）只是随机选择一个部分，而不是正确重建表达式
   - 这会导致交叉操作实际上退化为随机选择

3. **适应度函数权重配置可能不合理**
   - `ic_weight = 1.0`, `ir_weight = 2.0`，ICIR权重过高可能导致过度优化稳定性而忽略绝对IC值

### 1.2 因子表达式解析器评估

**✅ 正确部分：**
- 支持丰富的算子：时序算子、截面算子、数学算子、二元算子、逻辑算子
- 使用 `eval()` 配合安全的命名空间（namespace）执行
- 自动处理行业中性化函数的移除
- 支持多种数据字段和衍生字段计算

**⚠️ 潜在问题：**

1. **安全性问题**：
   - 使用 `eval()` 和 `exec()` 执行用户输入的公式字符串
   - 虽然限制了命名空间，但仍存在潜在的安全风险

2. **性能问题**：
   - 每次计算都重新解析表达式，没有缓存机制
   - 对于大量因子评估，效率较低

3. **错误处理不够精细**：
   - 所有异常都被捕获并返回-999，不利于调试

### 1.3 ICIR计算逻辑评估

**✅ 正确部分：**
- 使用Rank IC（Spearman相关系数），这是量化领域的标准做法
- 按日期计算截面IC，然后统计均值和标准差
- 最少股票数限制（min_stocks=10），避免小样本偏差

**⚠️ 关键问题 - Label定义：**

**问题发现：** 在 `ga_search.py` 中，`_safe_spearman_ic` 和 `_safe_ir` 函数直接使用传入的 `returns` 序列作为label。

**关键疑问：** 这个 `returns` 是：
1. 当期收益率？（T日）
2. 未来一期收益率？（T+1日）

**标准做法：** 在量化因子研究中，label应该是**未来N期收益率**（通常是T+1日收益率），这样才能评估因子的预测能力。

**需要检查：** `data_manager.py` 中如何生成 `returns` 序列，以及是否正确进行了时间对齐（shift操作）。
