## 核心痛点

1. 遗传算法完全手写
   - 没有使用DEAP库，自己实现了ExpressionTreeNode、SubtreeCrossover、锦标赛选择等
   - 但文件注释却写着"基于DEAP的遗传算法因子搜索"（名不副实）
   - 缺乏DEAP的成熟功能：Statistics统计、HallOfFame管理、并行评估等

2. 路径混乱，耦合严重
   - factor_library 反复修改 sys.path 导入 alpha_factor_test/clean
   - 两个目录都有 config.py、data_manager、ic_analyzer、model_trainer
   - 循环依赖风险高，维护困难

3. 重复代码多
   - IC计算逻辑在多处重复
   - 数据加载逻辑在多处重复
   - 配置参数分散在多个config文件

4. 中间文件泛滥
   - 几十个CSV/PKL中间产物散落在目录中
   - 没有明确的数据生命周期管理
   - 127MB的PKL文件不知是否可以删除

5. 无统一入口
   - 用户不知道该运行哪个脚本
   - batch_search.py、search_one.py、run_alpha_ga_search.py 功能重叠
   - CLI参数不一致

6. 数据源单一
   - 仅支持Qlib内置的日频数据
   - 不支持分钟级数据（1min/5min/15min）
   - 不支持基本面数据（PE/PB/ROE等）
   - 不支持外部特征导入（CSV/Parquet）

## 终极目标

构建一个统一的智能因子工厂（Smart Factor Factory），具备以下能力：

### 1. 多源数据导入层

支持的数据类型：
- 分钟级数据：1min/5min/15min K线（券商下载格式）
- 天级数据：日线OHLCV（标准格式）
- 基本面数据：PE/PB/ROE/营收/净利润等财务指标
- 自定义特征：CSV/Parquet格式的任意特征
- 外部数据源预留接口：Wind、Tushare、AKShare

数据管理功能：
- 统一数据格式：自动对齐到Qlib的MultiIndex(datetime, instrument)结构
- 数据质量检查：
  - 缺失值统计与填充策略
  - 异常值检测（3-sigma、MAD方法）
  - 时间连续性校验（交易日历对齐）
  - 截面数据完整性检查
- 数据版本管理：每次导入生成版本号，支持回滚
- 元数据记录：数据来源、导入时间、字段说明

### 2. 遗传算法因子搜索核心

基于DEAP库重构：
- 使用DEAP的creator定义个体和适应度
- 使用DEAP的toolbox注册算子（交叉、变异、选择）
- 使用DEAP的tools.HallOfFame管理历史最优
- 使用DEAP的tools.Statistics进行统计
- 使用DEAP的algorithms.eaSimple或自定义算法

多目标优化：
- IC最大化（预测能力）
- ICIR最大化（稳定性）
- 低换手率（交易成本）
- 低相关性（与已有因子库）

约束条件支持：
- 表达式复杂度上限（最大树深度、最大节点数）
- 算子白名单（允许使用的算子列表）
- 字段白名单（允许使用的数据字段）
- 自定义约束函数

种子注入：
- 用户可以提供优质种子表达式作为进化起点
- 支持从数据库导入历史优秀因子作为种子
- 支持从文件导入种子列表

无意义表达式过滤：
- 语法层检测：相同字段相减 (close-close)、纯常数表达式
- 语义层检测：因子值为常数（标准差接近0）、全NaN
- 运行时检测：除以零、对负数开方、对负数取log

结构去重：
- 提取结构签名：将所有数字替换为#（如 -20*ts_delta(...,1) → -#*ts_delta(...,#)）
- 等价因子检测：仅系数不同的因子视为等价
- 保留最优：同一结构中只保留fitness最高的因子

并行评估：
- 支持多进程并行计算因子IC/ICIR
- DEAP的tools.Pool支持
- 可配置并行度

### 3. 自动回测与入库

因子评估指标：
- IC（Pearson相关系数）
- ICIR（IC均值/IC标准差）
- RankIC（Spearman相关系数）
- RankICIR
- IC胜率（IC>0的比例）
- 分组收益（5分组，多头组、空头组收益）
- 多空收益（多头-空头）
- 超额收益（多头-平均）

自动入库规则：
- 阈值条件：ICIR>0.5, |IC|>0.02, IC胜率>50%（可配置）
- 结构去重入库：相同语法结构的因子只保留ICIR最优的一个
- 元数据记录：搜索时间、评估区间、GA参数、种子来源
- 标签系统：自动打标（如"GA生成"、"种子衍生"、"高频"等）

因子筛选作为特征批次回测：
- 通过因子筛选作为特征批次回测
- 统一回测对比
- 类似Alpha50对齐研究的功能：所有因子除了特征表达式不同外，其他设置严格对齐
  - 相同样本区间
  - 相同股票池
  - 相同数据预处理
  - 相同Label计算公式
  - 相同评估指标

基准对齐：
- 确保公平对比：除因子表达式外，所有配置一致，需要有测试脚本进行严格测试，样本对齐各项设置对齐。
- 可配置基准因子（如Alpha158、市场因子）
- 相对绩效评估

### 4. 可视化与因子管理

Web界面（Streamlit）：

1. 因子列表页
   - 表格展示所有因子
   - 支持按IC/ICIR/RankICIR/IC胜率/分组收益排序
   - 筛选：标签过滤、阈值过滤、时间范围过滤
   - 批量操作：选择多个因子进行对比、导出、删除

2. 因子详情页
   - 基本信息：因子ID、表达式、创建时间、标签
   - 核心指标：IC、ICIR、RankIC、RankICIR、IC胜率
   - 分组收益：5分组收益柱状图
   - IC时序图：IC值随时间变化曲线
   - 因子值分布：直方图、箱线图
   - 元数据：搜索参数、评估区间、种子来源

3. 因子对比页
   - 勾选2-4个因子
   - IC序列对比图（多线）
   - 分组收益对比图（分组柱状图）
   - 相关性矩阵热力图
   - 累计收益曲线

4. 因子管理页
   - 批量删除
   - 标记最优
   - 设置标签
   - 导出CSV/Excel
   - 导入因子

时序观测：
- 支持查看因子在不同时期的表现差异
  - 训练期 vs 验证期 vs 测试期
  - 不同年份表现
  - 滚动窗口IC变化
- 衰减分析：因子IC随时间衰减的趋势可视化
- 稳定性评估：IC波动率、最大回撤

# src/ga/searcher.py
from deap import base, creator, tools, algorithms

class GAFactorSearcher:
    """基于DEAP的遗传算法因子搜索器"""
    
    def __init__(self, engine: AlphaEngine, config: GAConfig):
        self.engine = engine
        self.config = config
        self.toolbox = base.Toolbox()
        self._setup_deap()
    
    def _setup_deap(self):
        """配置DEAP环境"""
        # 创建适应度类
        creator.create("FitnessMax", base.Fitness, weights=(1.0,))
        # 创建个体类
        creator.create("Individual", ExpressionTree, fitness=creator.FitnessMax)
        
        # 注册算子
        self.toolbox.register("evaluate", self._evaluate_fitness)
        self.toolbox.register("mate", self._crossover)
        self.toolbox.register("mutate", self._mutate)
        self.toolbox.register("select", tools.selTournament, tournsize=3)
    
    def search(
        self, 
        seed_expressions: Optional[List[str]] = None,
        existing_factors: Optional[List[pd.Series]] = None
    ) -> List[FactorResult]:
        """执行遗传算法搜索"""
        # 初始化种群
        population = self._initialize_population(seed_expressions)
        
        # 创建Hall of Fame
        hof = tools.HallOfFame(maxsize=50)
        
        # 创建统计
        stats = tools.Statistics(lambda ind: ind.fitness.values)
        stats.register("avg", np.mean)
        stats.register("max", np.max)
        stats.register("min", np.min)
        
        # 执行进化
        population, logbook = algorithms.eaSimple(
            population, self.toolbox,
            cxpb=self.config.crossover_prob,
            mutpb=self.config.mutation_prob,
            ngen=self.config.n_generations,
            stats=stats,
            halloffame=hof,
            verbose=True
        )
        
        return self._extract_results(hof)
